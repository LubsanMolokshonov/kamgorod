# Подготовка и выполнение релиза (Agent Team)

Ты — оркестратор релиза для проекта fgos.pro. Координируй аудит, деплой и верификацию.

## Алгоритм

### Фаза 1: Аудит

Запусти **general-purpose agent** с задачей:

1. **Security-аудит** — выполни проверки из `/check-security`:
   - SQL-инъекции в новых/измененных файлах
   - XSS в pages/ и includes/
   - CSRF в ajax/
   - Debug-код (var_dump, print_r, die)
   - Hardcoded credentials

2. **Git-статус** — проверь:
   - `git status` — нет ли незакоммиченных изменений
   - `git log --oneline origin/main..HEAD` — какие коммиты войдут в релиз
   - Нет ли конфликтов

3. **Миграции** — проверь:
   - Все ли миграции корректны (нет синтаксических ошибок в SQL)
   - Правильная ли нумерация (нет пропусков/дублей)
   - FK ссылаются на существующие таблицы

4. **PHP-синтаксис** — проверь все измененные PHP-файлы:
   ```bash
   git diff --name-only origin/main..HEAD -- '*.php' | xargs -I {} php -l {}
   ```

**Если найдены CRITICAL-проблемы — ОСТАНОВИ релиз и сообщи пользователю.**

### Фаза 2: Подтверждение

Покажи пользователю сводку аудита:
- Результаты security-проверки
- Список коммитов для деплоя
- Есть ли миграции для применения
- Спроси подтверждение через AskUserQuestion

### Фаза 3: Деплой

Запусти **general-purpose agent** с задачей деплоя (логика из `/deploy`):

1. Создай бэкап на сервере
2. Загрузи измененные файлы через SCP
3. Примени миграции
4. Установи права доступа

Конфигурация:
- Сервер: `root@141.105.69.45`
- Путь: `/var/www/html`
- Авторизация: `sshpass -p '1uf_d7C23o' ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null`

### Фаза 4: Верификация

Запусти **general-purpose agent** с задачей:

1. **Доступность ключевых страниц** — curl-проверка (ожидаем HTTP 200):
   ```
   https://fgos.pro/
   https://fgos.pro/konkursy/
   https://fgos.pro/vebinary/
   https://fgos.pro/kursy/
   https://fgos.pro/zhurnal/
   https://fgos.pro/olimpiady/
   ```

2. **API/Webhook** — проверка:
   ```
   POST https://fgos.pro/api/webhook/yookassa.php → 403 (OK)
   GET https://fgos.pro/api/check-payment.php?order_number=test → "Order not found" (OK)
   ```

3. **Логи Apache** — проверь на ошибки:
   ```bash
   ssh ... "tail -30 /var/log/apache2/error.log | grep -i 'fatal\|error' | tail -10"
   ```

4. **PHP-ошибки** — проверь:
   ```bash
   ssh ... "tail -20 /var/www/html/logs/*.log 2>/dev/null"
   ```

## Правила

- **CRITICAL-проблемы останавливают релиз** — не продолжай деплой
- Всегда делай бэкап перед деплоем
- Не деплой .env без явного подтверждения
- Если сайт не отвечает после деплоя — немедленно сообщи
- Если миграция упала — сообщи и предложи rollback
- Логи проверяй сразу после деплоя, не откладывай
