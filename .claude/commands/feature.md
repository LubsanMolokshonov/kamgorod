# Создание новой фичи (Agent Team)

Ты — оркестратор для создания новой фичи в проекте fgos.pro. Координируй работу sub-agents для создания полного набора файлов.

## Входные параметры

Пользователь описывает фичу в свободной форме. Примеры:
- "Добавь раздел методических материалов"
- "Создай систему отзывов"
- "Добавь раздел вакансий для педагогов"

## Алгоритм

### Фаза 1: Планирование

Запусти **Explore agent** для анализа:

1. Изучи существующие аналогичные разделы (competitions, webinars, courses) как образцы
2. Определи, какие файлы нужно создать:
   - Миграция (таблица БД)
   - Класс бизнес-логики
   - Страница-листинг
   - Детальная страница
   - AJAX-эндпоинты
   - CSS и JS
   - Роуты в .htaccess
   - SEO-маппинги

3. Покажи пользователю план и спроси подтверждение:
   - Список файлов для создания
   - Структура таблицы БД
   - URL-маршруты

### Фаза 2: Backend

Запусти **general-purpose agent** с задачей:

1. **Миграция** — создай файл в `database/migrations/`:
   - Определи следующий номер миграции
   - CREATE TABLE с нужными колонками
   - Индексы на slug, is_active, FK
   - `ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci`

2. **Класс** — создай `classes/{ClassName}.php`:
   - Прочитай `classes/Competition.php` как образец
   - `private Database $db` + конструктор с `$pdo`
   - Методы: `getAll()`, `getBySlug()`, `getById()`, `create()`, `update()`
   - Prepared statements с `?` placeholders

3. **AJAX** — создай эндпоинты в `ajax/`:
   - Прочитай существующие AJAX как образец
   - Validator + JSON-ответ + try/catch

### Фаза 3: Frontend

Запусти **general-purpose agent** с задачей:

1. **Страница-листинг** — `pages/{slug}.php`:
   - Прочитай `pages/competitions.php` как образец
   - header/footer, SEO-мета, $additionalCSS/$additionalJS
   - Фильтрация и пагинация (если нужно)

2. **Детальная страница** — `pages/{slug}-detail.php`:
   - Прочитай `pages/competition-detail.php` как образец
   - JSON-LD structured data
   - Breadcrumbs

3. **CSS** — `assets/css/{slug}.css`:
   - Адаптивный, мобильный-first
   - Фиолетовая тема (#6c5ce7)

4. **JS** — `assets/js/{slug}.js`:
   - jQuery-паттерны
   - AJAX-вызовы

### Фаза 4: Интеграция

Запусти **general-purpose agent** с задачей:

1. **Роутинг** — обновить `.htaccess`:
   - RewriteRule для листинга и детальной страницы
   - Вставить в правильное место (прочитай текущий .htaccess)

2. **SEO URL** — обновить `includes/seo-url.php`:
   - Добавить маппинг для нового раздела

3. **Sitemap** — обновить `sitemap.php`:
   - Добавить новый раздел в генерацию

4. **Навигация** — обновить `includes/header.php`:
   - Добавить ссылку в меню навигации

5. **SEO-проверка** — выполнить проверки из `/check-seo`:
   - Title, Description, Canonical URL
   - JSON-LD
   - Sitemap

### Фаза 5: Отчет

Выведи итоговый отчет:
- Список созданных файлов
- Список измененных файлов
- Структура таблицы БД
- URL-маршруты
- Что нужно сделать вручную (если что-то)
- Команда для применения миграции: `php migrate.php`

## Правила

- Всегда читай существующие файлы как образцы перед созданием новых
- Следуй всем паттернам из CLAUDE.md
- Спрашивай подтверждение после фазы планирования
- Используй русский язык для интерфейса и комментариев
- Не создавай лишних файлов — только то, что реально нужно
