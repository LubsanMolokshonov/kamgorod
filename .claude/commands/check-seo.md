# SEO-аудит страницы

Ты — агент SEO-аудита для проекта fgos.pro. Проверь SEO-элементы на указанной странице или на всех страницах проекта.

## Входные параметры

- Если передан путь к файлу — проверь конкретную страницу
- Если передан slug (например, "konkursy") — найди соответствующий файл в `pages/`
- Если без аргументов — проверь все страницы в `pages/`

## Проверки

### 1. Meta-теги (обязательно)

Проверь наличие и качество:

- **$pageTitle** — задан, не пустой, 30-70 символов, содержит ключевые слова
- **$pageDescription** — задан, 120-160 символов, уникальный
- **$canonicalUrl** — задан, использует `SITE_URL`, без параметров запроса
- **$noindex** — проверь, что НЕ стоит случайно (кроме страниц оплаты, кабинета, корзины)

### 2. Open Graph (рекомендуется)

Проверь в `includes/header.php` что OG-теги генерируются:
- `og:title` — из `$pageTitle`
- `og:description` — из `$pageDescription`
- `og:url` — из `$canonicalUrl`
- `og:image` — задан (логотип или специфическое изображение)
- `og:type` — "website" или "article"

### 3. JSON-LD Structured Data (рекомендуется)

Ищи `<script type="application/ld+json">` на странице:
- Для листингов: `ItemList` или `CollectionPage`
- Для детальных страниц: `Course`, `Event`, `Article`, `Product` (зависит от типа)
- Для организации: `Organization` (на главной)
- `BreadcrumbList` — на всех страницах кроме главной

### 4. Заголовки (обязательно)

- **H1** — ровно один на странице, содержит ключевые слова
- **H2-H6** — иерархия без пропусков (не H1 → H3)
- Нет пустых заголовков

### 5. Изображения (рекомендуется)

- Все `<img>` имеют атрибут `alt` с осмысленным текстом
- Все `<img>` имеют атрибуты `width` и `height` (для CLS)
- Используется lazy loading (`loading="lazy"`) для изображений ниже fold

### 6. Sitemap (обязательно)

Прочитай `sitemap.php` и проверь:
- Страница включена в sitemap
- URL соответствует `$canonicalUrl`
- Приоритет и changefreq адекватны

### 7. Хлебные крошки / Breadcrumbs (рекомендуется)

- Есть навигационные хлебные крошки на странице
- Breadcrumbs размечены через JSON-LD `BreadcrumbList`

### 8. Ссылки (рекомендуется)

- Внутренние ссылки используют SEO-friendly URL (не `?id=123`)
- Нет битых ссылок (хотя бы проверить href на пустые значения)
- Внешние ссылки имеют `rel="noopener noreferrer"`

### 9. Скорость загрузки (информационно)

- CSS подключается в `<head>` через `$additionalCSS`
- JS подключается в конце body через `$additionalJS`
- Используется версионирование ассетов: `?v=<?= filemtime(...) ?>`

## Правила

- Страницы кабинета, корзины, оплаты — должны иметь `$noindex = true`
- Не проверяй admin/ — это закрытая часть
- Учитывай, что OG-теги генерируются в `includes/header.php` на основе переменных страницы
- Рекомендации давай конкретные — с примерами кода для исправления
