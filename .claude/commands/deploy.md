# Деплой на продакшн

Ты — агент деплоя для проекта fgos.pro. Выполни полный цикл деплоя на продакшн-сервер.

## Конфигурация

- Сервер: `root@141.105.69.45`
- Путь проекта на сервере: `/var/www/kamgorod/`
- Docker-контейнер: `pedagogy_web` (web), `pedagogy_db` (MySQL)
- Подключение: `ssh root@141.105.69.45` (по ключу)
- Репозиторий: `git@github.com:LubsanMolokshonov/kamgorod.git`

## Алгоритм

### Шаг 1: Проверка локального состояния

1. `git status` — проверь незакоммиченные изменения
2. Если есть изменения — спроси пользователя, нужно ли закоммитить и запушить
3. `git log --oneline origin/main..HEAD` — покажи, какие коммиты ещё не запушены
4. Если нет новых коммитов и нет изменений — сообщи, что деплоить нечего

### Шаг 2: Коммит и пуш (если нужно)

1. Если есть незакоммиченные изменения:
   - `git add` — добавить файлы (исключить `.env`, личные файлы)
   - `git commit` — с осмысленным сообщением на русском
2. `git push origin main`
3. Убедись, что пуш прошёл успешно

### Шаг 3: Синхронизация на сервере

1. Подключись к серверу: `ssh root@141.105.69.45`
2. Проверь состояние: `cd /var/www/kamgorod && git status`
3. Если на сервере есть локальные изменения — сохрани через `git stash`
4. Подтяни изменения: `git pull origin main`
5. Если был stash — восстанови: `git stash pop`
6. При конфликтах — **остановись и сообщи пользователю**

### Шаг 4: Миграции

```bash
ssh root@141.105.69.45 "docker exec pedagogy_web php /var/www/html/migrate.php"
```

- Покажи вывод пользователю
- Если есть критические ошибки (не warnings) — **останови деплой**

### Шаг 5: Перезапуск контейнера

```bash
ssh root@141.105.69.45 "docker restart pedagogy_web && sleep 3 && docker ps --filter name=pedagogy_web --format '{{.Names}}\t{{.Status}}'"
```

Убедись, что контейнер поднялся (статус `Up`).

### Шаг 6: Верификация

1. Проверь главную: `curl -s -o /dev/null -w "%{http_code}" https://fgos.pro/`  → ожидай 200
2. Проверь раздел курсов: `curl -s -o /dev/null -w "%{http_code}" https://fgos.pro/kursy/`  → ожидай 200
3. Проверь логи на ошибки:
```bash
ssh root@141.105.69.45 "docker logs pedagogy_web --tail 20 --since 1m"
```

### Шаг 7: Отчет

Выведи краткий отчет:
- Что было задеплоено (количество коммитов/файлов)
- Были ли применены миграции (сколько)
- Статус сайта (HTTP-коды)
- Был ли перезапуск контейнера
- Общий статус: УСПЕШНО / С ОШИБКАМИ

## Важные правила

- **Не коммить `.env`** — файл с секретами
- **Не деплой `vendor/`** — на сервере свой composer install
- **Сохраняй серверные изменения** через `git stash` перед pull
- **Останавливайся при ошибках миграции** — не продолжай
- **Останавливайся при конфликтах** — не резолви автоматически
- Если сайт не отвечает после деплоя — сразу сообщи пользователю
