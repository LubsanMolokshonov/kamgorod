# Деплой на продакшн

Ты — агент деплоя для проекта fgos.pro. Выполни полный цикл деплоя на продакшн-сервер.

## Конфигурация сервера

- Сервер: `root@141.105.69.45`
- Путь проекта: `/var/www/html`
- Авторизация: `sshpass -p '1uf_d7C23o' ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null`
- SCP: `sshpass -p '1uf_d7C23o' scp -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null`

## Алгоритм

### Шаг 1: Проверка состояния

1. Выполни `git status` — проверь, есть ли незакоммиченные изменения
2. Если есть незакоммиченные изменения — **предупреди пользователя** и спроси, продолжать ли
3. Выполни `git log --oneline origin/main..HEAD` — покажи, какие коммиты будут задеплоены
4. Если нет новых коммитов — сообщи и спроси, всё равно деплоить

### Шаг 2: Подтверждение

Спроси пользователя через AskUserQuestion:
- "Деплоить на продакшн?" с опциями:
  - "Да, деплоить" — продолжить
  - "Нет, отмена" — прервать

### Шаг 3: Определение файлов для деплоя

1. Собери список измененных файлов: `git diff --name-only origin/main..HEAD`
2. Также добавь untracked файлы, которые пользователь хочет задеплоить (спроси если есть)
3. **НИКОГДА не деплой `.env` без явного подтверждения пользователя**
4. Исключи из деплоя: `.git/`, `.claude/`, `docker-compose.yml`, `Dockerfile`, `*.md`, `deploy*.sh`, `node_modules/`, `vendor/`

### Шаг 4: Бэкап на сервере

```bash
sshpass -p '1uf_d7C23o' ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null root@141.105.69.45 "cd /var/www/html && tar -czf /root/backup_$(date +%Y%m%d_%H%M%S).tar.gz --exclude='vendor' --exclude='uploads' --exclude='.git' ."
```

### Шаг 5: Загрузка файлов

Для каждого файла из списка:
1. Создай директорию на сервере если нужно: `ssh ... "mkdir -p /var/www/html/$(dirname file)"`
2. Загрузи файл: `scp ... file root@141.105.69.45:/var/www/html/file`
3. Установи права: `chmod 644` для PHP/CSS/JS, `chmod 755` для директорий

### Шаг 6: Миграции

```bash
sshpass -p '1uf_d7C23o' ssh ... root@141.105.69.45 "cd /var/www/html && php migrate.php"
```

Покажи вывод пользователю. Если есть ошибки — **останови деплой** и сообщи.

### Шаг 7: Верификация

1. Проверь доступность сайта:
```bash
curl -s -o /dev/null -w "%{http_code}" https://fgos.pro/
```
Ожидаемый код: 200

2. Проверь логи Apache на ошибки:
```bash
sshpass -p '1uf_d7C23o' ssh ... root@141.105.69.45 "tail -20 /var/log/apache2/error.log"
```

3. Проверь webhook:
```bash
curl -s -o /dev/null -w "%{http_code}" -X POST https://fgos.pro/api/webhook/yookassa.php
```
Ожидаемый код: 403 (нормально — значит webhook работает, но отклоняет неавторизованные запросы)

### Шаг 8: Отчет

Выведи краткий отчет:
- Сколько файлов загружено
- Были ли применены миграции
- Статус сайта (HTTP-код)
- Есть ли ошибки в логах
- Общий статус: УСПЕШНО / С ОШИБКАМИ

## Важные правила

- **Не деплой .env без подтверждения** — это файл с секретами
- **Не деплой vendor/** — на сервере свой composer install
- **Всегда делай бэкап** перед загрузкой файлов
- **Останавливайся при ошибках миграции** — не продолжай деплой
- Если сайт не отвечает после деплоя — сразу сообщи пользователю
